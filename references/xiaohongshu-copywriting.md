# 小红书发布文案规范

前端术语卡的发布文案要短、快、好懂，让读者看到术语就知道它出现在哪里、和什么容易混淆。

## 固定结构

```text
标题：用户可感知的问题或术语解释

正文：
场景钩子（1 句）

📌 一句话理解：...
✅ 你会看到：...
✅ 它解决：...
⚠️ 它不等于：...

💡 记忆口诀：...
💾 适合收藏，之后遇到这个术语可以快速对照。

标签：5-10 个精准标签
```

## 写作要求

- 正文使用 4-8 个有语义的 emoji，优先放在分点开头。
- 至少使用 3 个分点，优先使用 `📌`、`✅`、`⚠️`、`💡`、`💾` 等符号。
- 正文保持 2-5 个短段，不要写成术语词典或长教程。
- 必须出现一个真实界面、用户行为或开发场景。
- 必须说清一个最容易混淆的术语；不要一次比较太多概念。
- 结尾提供记忆口诀、使用判断或收藏引导。
- 标签单独成行，使用 5-10 个术语、前端、产品和受众相关标签，不给标签加 emoji。

## 风格边界

- 可以口语化，但要明确术语语境；公司内部缩写不能说成统一 Web 标准。
- 不要用大量 emoji 代替定义和区别。
- 不要把标题、正文和标签混成一段。
