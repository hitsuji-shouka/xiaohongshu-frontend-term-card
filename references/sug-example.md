# 示例主题：SUG

## 术语定位

SUG 通常是 Suggestion 或 Query Suggestion 的简称，在搜索产品语境中常指“搜索建议”或“搜索联想”。它不是统一的 Web 标准术语，不同团队可能叫 suggest、autocomplete、query suggestion 或 search suggestion。

## 单页版

标题：`SUG 是什么？搜索框下面那排联想词`

核心内容：

- SUG = 搜索建议 / 搜索联想。
- 用户输入关键词时，搜索框下方动态出现候选词。
- 作用是降低输入成本，帮助用户明确搜索意图。
- 它不是固定 Select，因为用户仍可自由输入。

视觉隐喻：

- 小人正在搜索框输入“上海”。
- 搜索框下方伸出 4 张候选词卡片。
- 候选词卡片标注“上海天气”“上海迪士尼”“上海租房”。
- 一侧用小标签写“自由输入”，另一侧写“动态建议”。

## 双页版

### 第 1 页：定义

```text
SUG 是搜索建议。
当你还没输完，系统先猜猜你可能想搜什么。
```

必须准确表达：

- SUG 是搜索产品语境中的常见缩写。
- 候选词会随输入变化。
- 用户可以点候选词，也可以继续输入。

### 第 2 页：流程和区别

流程：

```text
输入关键词 → 防抖 → 请求候选词 → 召回/排序 → 展示下拉建议 → 点击或继续输入
```

与 Select 的区别：

- SUG：辅助自由输入，候选词动态变化。
- Select：从已有固定选项中选择。

实现注意点：

- 输入防抖。
- 旧请求不要覆盖新结果。
- 支持键盘上下选择、Enter 确认、Esc 关闭。
- 注意中文输入法 composition 事件。
- 关注可访问性，候选词列表应能被键盘和屏幕阅读器理解。
