# 配图质量门槛

前端术语卡最容易被生成为“很多空框 + 少量文字”。目标是做一张像之前完整案例一样的知识卡，而不是线框图或 UI 组件素材板。

## 构图硬规则

- 每页只保留一个主视觉动作，配 2-3 个有内容的辅助模块。
- 页面最多 4 个边框模块；每个模块都必须有真实文字、界面状态或结论，禁止空卡片、占位横线、灰色假文本和无意义虚线框。
- 第 1 页优先使用一个真实界面场景：搜索框、Toast、Modal 等必须“正在发生”，不要拆成三张空卡片。
- 第 2 页优先使用一条 4-5 步流程，再配一个紧凑对比区和一个注意事项区；不要使用九宫格、三列空框或复杂表格。
- 标题区独立占顶部 12%-16%，标题、副标题和页码不能压到插图区或互相重叠。
- 留白只用于分隔内容，不得出现大块“准备放内容但没有内容”的空白。

## SUG 特别规则

页面应呈现：输入关键词、候选词动态出现、用户继续输入或点击选择。候选词卡片必须有真实短词，例如“上海天气”“上海地铁”，禁止用灰色横线代替文字。

不要同时放三张独立搜索卡、三组空白解释卡和多个装饰箭头。优先一个大搜索界面 + 2 个解释 callout。

## 生图提示词必须包含

```text
finished Xiaohongshu term card, not a UI wireframe,
one realistic interface scene, no empty boxes, no placeholder lines,
no blank cards, no overlapping title, no clipped Chinese labels,
2-3 compact callouts with real content, clear reading order,
purposeful whitespace, rich but uncluttered teaching card
```

中文长句和精确术语优先用可控的后期排版；如果当前环境不能做后期排版，就减少图中文字并重新生成，不能交付有错字、重叠或占位线的图片。

## 生成后拒绝条件

出现以下任一情况，必须重新生成或重新排版：

- 标题压线、越界、重叠或被装饰遮挡。
- 出现空白卡片、灰色占位线、假搜索结果或空表格。
- SUG 候选词不是实际文字，或 SUG 被画成固定 Select。
- 中文、术语或流程标签乱码、重复或截断。
- 画面只有 UI 素材，没有解释用户行为和术语含义。
- 页面一眼看不出“用户做了什么、系统发生了什么”。
