# 前端术语卡模板

## 术语解释卡字段

```text
英文术语：
常见中文名：
所属层级：
一句话定义：
解决的问题：
用户看到的表现：
关键流程：
易混淆术语：
核心区别：
实现注意点：
可访问性提醒：
视觉隐喻：
```

## 所属层级

- UI 组件术语：Toast、Modal、Drawer、Tooltip、Skeleton。
- 搜索与产品术语：SUG、Query、召回、排序、曝光、CTR。
- 前端工程术语：CSR、SSR、SSG、Hydration、BFF、Micro Frontend。
- 编程技巧术语：Debounce、Throttle、Lazy Load、Virtual List。

## 内容原则

- 默认 1-2 页。
- 一页不超过一个核心流程和一个对比关系。
- 先讲“用户看到什么”，再讲“前端怎么做”。
- 对非标准缩写，要说明采用的语境。
- 不放长代码，只放实现关键词。

## 视觉原则

- 优先画真实界面发生的事。
- 把 UI 组件画成页面里的行为，而不是独立图标。
- 角色要输入、点击、选择、等待或接收反馈。
- 对比相似术语时，用两个并排小场景，不写长表格。
